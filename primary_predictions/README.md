# Primary predictions

This folder consumes the products generated by `primary_fits/`.

Run the fits first:

```bash
python primary_fits/run_primary_fits.py
```

Then generate primary prediction tables and bands:

```bash
python primary_predictions/run_primary_predictions.py
```

For the dedicated low-pressure IR scan starting at 0.1 mbar:

```bash
python primary_predictions/run_primary_ir_low_pressure_predictions.py
```

For the generic overlay/multibar plots, including the Ar--CF4 IR 1--5 bar overlay:

```bash
python primary_predictions/run_primary_multiband_predictions.py
```

The ordinary runner also calls these overlays by default. It also generates
the individual Ar--CF4 IR 1, 2, 3, 4 and 5 bar PDFs. Disable overlays from
Python with `main(make_multibands=False)`.

Outputs:

- `data/Predictions/primary_selected_yields_common_arcf4_norm.csv`
- `data/Predictions/primary_selected_yields_own_norm.csv`
- `data/Predictions/Bands/*.csv`
- `data/Tables/primary_selected_yields_*.tex`
- `primary_predictions/plots/primary_bands/*.pdf`

Band CSVs are cached.  If a band CSV already exists, the runner reuses it
unless `overwrite_bands=True` is passed from Python.

The two selected-yield tables differ only in normalization convention:

- `common_arcf4_norm`: all rows are divided by the central Ar--CF4 primary
  normalization.  This is useful for a single TFM-scale comparison table.
- `own_norm`: every gas/channel is divided by its own fitted normalization.
  This is useful for checking model-to-model consistency.


Multiband overlays
------------------

The plotting layer now has two levels:

- `BandCurveConfig`: one evaluated curve/band.  It carries its own `fit_name`,
  `component`, `pressure`, `x_grid` and `normalization`.  It can also point
  to a canonical `paper_overlay_id` so X-ray points are drawn with the same
  normalization logic used by the original paper-style single-band plots.
- `MultiBandPlotConfig`: one figure containing any number of `BandCurveConfig`
  objects.  The curves do not need to come from the same fit, gas or pressure.

For example, a future mixed plot can be built with curves such as
`ArN2_primary` at 1 bar, `ArXe_primary` at 3 bar and `HeCF4_primary` at 5 bar,
as long as the corresponding adapters exist in `PRIMARY_ADAPTERS`.

The Ar--CF4 IR 1--5 bar overlay is defined in `primary_multiband_plots()` and
writes:

- `data/Predictions/Bands/ArCF4_IR_primary_total_1p0bar.csv`
- `data/Predictions/Bands/ArCF4_IR_primary_total_2p0bar.csv`
- `data/Predictions/Bands/ArCF4_IR_primary_total_3p0bar.csv`
- `data/Predictions/Bands/ArCF4_IR_primary_total_4p0bar.csv`
- `data/Predictions/Bands/ArCF4_IR_primary_total_5p0bar.csv`
- `primary_predictions/plots/primary_bands/multibar_ir/ArCF4_IR_primary_total_1to5bar_overlay.pdf`
- `primary_predictions/plots/primary_bands/multibar_ir/ArCF4_IR_primary_total_1p0bar_bands.pdf`
- `primary_predictions/plots/primary_bands/multibar_ir/ArCF4_IR_primary_total_2p0bar_bands.pdf`
- `primary_predictions/plots/primary_bands/multibar_ir/ArCF4_IR_primary_total_3p0bar_bands.pdf`
- `primary_predictions/plots/primary_bands/multibar_ir/ArCF4_IR_primary_total_4p0bar_bands.pdf`
- `primary_predictions/plots/primary_bands/multibar_ir/ArCF4_IR_primary_total_5p0bar_bands.pdf`

For the default Ar--CF4 IR 1--5 bar overlay, each curve uses
`paper_overlay_id="ArCF4_IR_primary_total"`, so the X-ray points for the
corresponding pressure are read from `data/Experimental/ArCF4/csv/*.csv`, summed
over the IR lines, divided by the appropriate W value and normalized with
`ARCF4_PRIMARY_NORM`.

The secondary prediction machinery should use the same runner/band/table layer.
Its Garfield-specific choices should be expressed with `SecondarySelection`:
pressure/gap/electric-field/npe cuts live in config objects, not in plotting code.
